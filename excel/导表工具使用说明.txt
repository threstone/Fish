导表工具使用说明:
1.	首先检出 我们的程序配置目录表到任意文件目录地址为: svn://subversion.coding.net/SunYoung/xiuzhenconfig
2.	打开ExcelExport 导表工具 在导出目录填上检出的目录路径
3.	关闭工具再打开导表工具 配置 的导出目录 会被写入到config.txt 
4.	勾选excel文件列表框选择要执行导出的文件。
5.	点击导出即可。

Excel格式说明:
1.导出工具会 选中导出的xlsx文件 遍历所有的页签进行导出。 页签内的数据会不会被导出取决于B1单元格的内容 。当内容为 base 、tiny  时此页签将会被导出。
2.当页签已经被命中为导出后 B2为 导出后的文件名。导出的配置分为 客户端和服务端 分别使用 json格式 和lua 格式。存放在config/data/client 和  config/data/server下。 config 代指你检出的配置路径。
3.B3为key的数量 。此值只在 B1(导出类型) 为base 时候有效(大部分 配置都 会为base)。 代指生成的数组 维度 . 同时数组的索引key 为 从B列 7行顺序增加。需要与程序确定需要什么样的配置 。

key的字段内容是禁止重复的。

比如当一个数据 为 
 id 	index 	value
 1	  1 	 1 
 1	   2	 2
 1	   3            3
 2	   1            4
 2	   2	 5
 2	   3	 6

当导出的key 为 1 时 生成的配置内容是生成不了 的。因为 id 重复了。

这种情况下 就 需要改key 为 2
最终生成的数据是 以lua 为例。
{
  [1] = {[1]={value = 1},[2]={value=2},[3]={value=3}},
  [2] = {[2]={value = 4},[2]={value=4},[3]={value=6}},
}

4.tiny  仅仅生成key value 的一种模式 只能有两个字段 一个key 一个value
例如有这样一个数据  表
 id 	value
 WEAPON	  1 
 DRESS	  2
将生成
{
   WEAPON = 1,
   DRESS = 2，
}

5.E1 E2 为导出文件的头部和尾部附加内容。 通常用于写文件表名称和结尾。方便程序调用。

6.第4行为注释字段可在此随意写注释。
5.第5行为每个单元格的中文 注释。
7.第6行为每个导出字段会被导出的配置 s 表示服务端 c表示客户端 sc 表示 客户端服务端都会导出。对于一些字段只需要服务端使用或者只需要 客户端使用的 根据需求配置。减少配置文件的大小。
8.第一行为每个导出字段的名称（不能使用中文 ）
9.第八行就是数据的 开始。
